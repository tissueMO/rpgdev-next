標準スクリプト仕様

プリセットデータに含まれる標準システムのスクリプトについて解説しています。
独自システムへの改変等で参考にして下さい。
なお、スクリプトの引数には型指定を行う構文がありませんが、引数名の頭に int/str/bool/func 等のプレフィックスを付けてその型を示すようにしています。
空の引数は構文エラーとなりますので、例えば文字列型で空の値を入れる場合は "" と指定するようにして下さい。

プレフィックスのない引数について

クラス型や名前から型が明らかであるもの (idx 等) については、作者の判断で省略しているものがあります。
与えるべき型が分からないときは、その関数が使われている箇所を検索してみましょう。

タイトルシーン

概要

  • マップシーンへと接続するシーンです。
  • マップに移動する前に、プレイ時間の計測に必要となるゲームの開始時刻を記憶しています。

階層構造

  • はじめから … 最初のマップへ移動します。
  • つづきから
    • セーブデータ選択 … セーブデータを読み込んでマップとイベントを復元します。Dキーで削除、Cキーでコピー（複写）します。
      • 複写モード：複写先選択
  • ゲーム終了

マップシーン

概要

  • マップシーンに入るとき、パーティの先頭者のグラフィックをセットします。
  • マップ間移動をしたとき、画面左上にマップ名を示すパネルを一定時間表示します（疑似スレッド）。
  • プレイヤーの移動操作が発生したとき、その方向に対しての移動判定および接触することで起動するイベントの判定を行います。
  • プレイヤーが1歩分歩き終えたところでエンカウント判定を行います。エンカウント係数を考慮した最低歩数に達するまでは発生しません。
  • プレイヤーが1歩分歩き終えたところで各パーティメンバーが罹患している間接効果の個別処理を行います。
  • プレイヤーがマップ外に出てしまった場合は、マップのループ設定に従って補正します。
  • 現在プレイヤーがいる座標のタイルがグループ番号 99番 であるとき、梯子属性とみなして北向きに固定します。
    この梯子属性はイベントには適用されません。

操作

  • イベント実行中は入力操作を受け付けません。
  • 主人公の移動操作は、上下左右キーの中で最後に押された方向に進みます。
  • 決定キーが押されたとき、決定キーによって起動するイベントの判定を行います。
  • キャンセルキーが押されたとき、メニューシーンへ移行します。
  • Aキーを押すことで速度が1段階増加します。
  • デバッグモードのとき、Sキーを押すことですり抜け有効＋エンカウント無効にできます。
  • F12キーによるリセットが有効です。

メニューシーン

概要

  • メニューシーンに入るとき、パーティメンバーのグラフィックをすべて読み込みます。
  • メニューシーンが終わるとき、パーティメンバーのグラフィックをすべて破棄します。
  • メニューの選択項目に応じた描画・入力処理を行っています。

  • アイテム

      • 対象メンバー選択
      • 場所選択
  • 装備
    • パーティメンバー選択
      • 装備大種別選択
        • 装備品選択 … アドオンによって装備が制限されていることがありますのでご注意下さい。
  • スキル
    • パーティメンバー選択
      • 習得スキル選択 … アドオンによって使用が制限されていることがありますのでご注意下さい。
        • 対象メンバー選択
        • 場所選択 … プリセットのユーザーDB「場所リスト」を参照します。
  • 並び替え
    • パーティメンバー選択
      • 移動先選択
  • セーブ
    • セーブデータ選択 … 上書きの確認は行いません。Dキーで削除、Cキーで複写（コピー）します。

  • 環境設定
    • 設定項目選択/設定変更 … 左右キーで設定値を変更します。

バトルシーン

概要

  • 戦闘に入るときに行うトランジションは、バトルシーンを呼び出す関数側で行います。
  • 同名のエネミーがいる場合は A, B, ... とナンバリングします。
  • ダメージを受けたとき、パーティキャラはウィンドウの振動、エネミーはグラフィックの点滅を行います。
  • 画面構成はいわゆるフロントビューで、味方のコマンドをまとめて選択した後にアクションが始まるターン制を採用しています。
  • パーティキャラやエネミーのような戦闘員単体を「アクター」と呼びます。
  • 「逃げる」はアクター個別に選択するようになっており、逃げる人数が多いほど成功確率が高くなります。成功した場合は戦闘が中断します。
  • 行動順番は原則として敏捷性の高い順に行われますが、一部のコマンドでは調整が行われます。

フェーズ

  • 戦闘開始前 … 暗転した画面の中でセットアップを行います。
  • コマンド選択 … 自律戦闘を行うキャラはコマンド選択ができません。全員のコマンド選択が終わると行動順番が決定されます。
  • 各アクター行動 … 行動順番に従って行動を行います。ダメージが発生すると死亡判定が行われ、各行動が終わると戦闘終了判定が行われます。
  • 戦闘終了後
    • 勝利 … 倒したエネミーの経験値とお金を獲得します。
      • レベルアップ
      • レベルアップに伴うスキル習得
      • 拾得アイテム
    • 敗北 … ゲームオーバーとなりタイトル画面に戻ります。
    • 逃走 … 戦闘を中断します。途中まで倒したエネミーの経験値やお金は獲得できません。

自律戦闘AI

  • パーティキャラもエネミーも同様の操作で実装します。
  • 初期状態は行動内容データの配列に既定の行動（何も行動しない）が一つ格納されている状態です。
  • 既定の行動を変更もしくは置き換える形で行動内容を設定します。
  • 行動配列の要素数を増やすことで複数行動ができます。ただし、連続性（同じアクターが必ず続けて行動すること）は保証されません。
  • エネミーにMPの概念は無く、スキルによってMPを消費しません。

計算式

  • 命中率 … アクター個別変数で管理しており、基本値は 100% です。さらに各行動の成功率も掛け合わせます。
  • 物理攻撃量 ＝ 基本値 + 行動者の物理攻撃力 / 2 - 対象者の物理守備力 / 4 * 属性耐性 + 分散値
  • 魔法攻撃量 ＝ 基本値 + 行動者の魔法攻撃力 / 2 - 対象者の魔法守備力 / 4 * 属性耐性 + 分散値
  • 回復量 ＝ 基本値 + 分散値
  • クリティカルヒット確率 ＝ 一定確率
  • クリティカルヒット時はダメージ量を2倍にして、対象アクターの守備力や防御を無視します。
  • 逃走確率 ＝ 100 / 生存人数 * 逃走人数 - 一定失敗確率 / (生存人数 - 逃走人数 + 1)

間接効果による処理の割り込み

基本的にはメッセージ表示を行います。
行動が関わるタイミングでは、その行動内容のオブジェクトが渡されます。
このオブジェクトを操作することでアクターの行動内容を後から差し替えることができるようになっています。

  • 罹患時
  • 解除時
  • ターン開始時
  • ターン終了時 … 「毒」はここでダメージ処理を行います。
  • アクター行動前 … 「混乱」はここで行動内容を強制的に「攻撃」に変更し、対象を敵味方問わずランダム選択します。「沈黙」はここでスキルを使用しようとしていたときに行動を無効化します。

階層構造（コマンド選択）

  • 攻撃 … 持っている武器で攻撃します。
    • 対象エネミー選択
  • 防御 … 受けるダメージを半減させます。
  • スキル … 習得しているスキルを使用します。
    • スキル種別選択
      • 習得スキル選択 … アドオンによって使用が制限されていることがありますのでご注意下さい。
        • 対象エネミー選択
        • 対象メンバー選択
  • アイテム … 所持しているアイテムを装備・使用します。
    • 所持品・装備品選択
      • 所持アイテム選択 … 使用または装備します。アドオンによって装備が制限されていることがありますのでご注意下さい。
        • 対象エネミー選択
        • 対象メンバー選択
      • 装備品選択 … 使用または解除します。アドオンによって装備の解除が制限されていることがありますのでご注意下さい。
        • 対象エネミー選択
        • 対象メンバー選択
  • 逃げる
