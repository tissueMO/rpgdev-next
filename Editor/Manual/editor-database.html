データベース エディター

本ソフトでは、データベースのことをDBと略します。
これらは一部を除いて表形式のコントロール上で編集します。
ユーザー定義データベースを除く組み込みデータベースは予め列が定義されており、形式を変更することはできません。

全般

IDについて

データベースには一般的に、項目を特定するための一意の識別番号（プライマリーキー）という概念があります。
本ソフトでは、これに相当するものを「FixedID」と呼び、システムが自動で割り振ります。
FixedIDは一度採番されると後から変更することはできませんので、既存の行を削除して別の行に同一のデータを復元したとしても、それらは別なデータであると認識されます。

プリセットデータについては、スクリプトとも関連していることもあるため、削除やコピーの操作にはくれぐれもご注意下さい。
項目名が $ から始まるものは削除やレコードの置き換えを行うと影響があります。
ただし、項目名が $ から始まっていないものについても、ゲーム画面に表示されてしまう都合で $ を付けられていないデータが一部存在します。

また、既定ではFixedIDの列は表示されず、左端にはIDという列があります。
この「ID」は、FixedIDのことではなくユーザーによって自由に与えられる便宜的な値のことです。
これは列単位で並び替えを行うときの順序を示す値であって、識別番号とは異なりますのでご注意下さい。

インポート・エクスポート

外部のプロジェクトからエクスポートした（書き出した）データベースをインポートする（取り込む）ことができます。
万が一FixedIDが衝突した場合は、自動的に調整されます。
インポートに伴って手動で取り込む等したスクリプトに変更前のFixedIDを直接参照している項目が含まれている場合、変更になった箇所はすべて書き換える必要がありますのでご注意下さい。

行操作

行に対する操作です。コピー、削除、アンドゥ、リドゥに対応しています。
コピーして生成される行には新たなFixedIDが設定されます。
環境設定にてFixedIDを表示しない設定になっている場合は、コピーした行と元の行の区別がつきませんので、スクリプトにて直接参照されている項目を誤って変更しないようにご注意下さい。

列操作

列に対する操作です。追加、編集、削除、左右移動に対応しています。
これらの操作はユーザーデータベースでのみ有効です。
列を編集する場合、列の型が変更されることもあるため、列の編集を確定するときにその列に含まれるセルのデータがすべて削除されます。

検索機能

指定した名前に一致する行を検索します。
検索にヒットした行が選択された状態になり、連続的に実行できます。
一つのデータベースに複数のサブデータベースが含まれるものにも対応しています。
ただし、ユーザー定義データベースについてのみ、データベース名の検索を行う挙動をします。

アドオン

アクター（パーティキャラ・エネミー）に関わる設定を動的に変更するための拡張的な機能です。
パーティキャラ・エネミー・クラス・装備アイテム・間接効果に搭載されており、最終的に紐づくアクターに対して適用されます。
ただし、クラスはそれ自体にパラメーターを倍率で変動させる機能があるため、パラメーターを増減させることはできません。
なお、アドオンの優先順位は低い順に ｛アクター ＜ クラス ＜ 装備アイテム ＜ 間接効果｝ となります。

属性耐性を設定 指定した属性の耐性を倍率で変動させます。
パーセント値で示すため 100 が等倍となります。
間接効果耐性を設定 指定した間接効果の耐性を倍率で変動させます。

有効にするときは 1 (TRUE) を、解除するときは 0 (FALSE) を指定します。
この設定が有効であるとき、装備も解除もできなくなります。

有効にするときは 1 (TRUE) を、解除するときは 0 (FALSE) を指定します。

有効にするときは 1 (TRUE) を、解除するときは 0 (FALSE) を指定します。

習得していない場合、この設定は無効となります。

列種別

セルを選択した状態で F2 キーを押すことで列の編集が開始されます。
なお、各列はエディター側で予め定義していますが、これらのデータの用途のほとんどがスクリプトに依存することになります。
そのため、スクリプトの実装によっては意味合いが変わる可能性があることにご注意下さい。

FixedID この列は環境設定を変更しない限りは非表示です。
このセルの値は自動的に割り振られるため、ユーザーによって変更することはできません。
単一値 単一の整数値が有効な列です。
有効範囲が指定されている場合はその範囲内の値のみ有効です。
複数値 複数の整数値が有効な列です。値同士を ; で区切ってセットします。
有効範囲が指定されている場合はその範囲内の値のみ有効です。
文字列 文字列が自由に入力できる列です。
長さに上限が指定されている場合はその範囲内の長さで有効です。
論理値 ON/OFFを切り替えられる列です。
プログラム側では ON=true/OFF=false として扱います。
既存リスト 他のデータベースを参照するリストから項目を一つだけ選択できる列です。
選択した項目のFixedIDが単一値として保存されます。
独自リスト 単純なリストから項目を一つだけ選択できる列です。
選択したインデックス（0 から始まる）が単一値として保存されます。
組み込みデータベースでは、スクリプトに公開している組み込みの列挙体にそのインデックスが定義されています。
IDリスト 他のデータベースのIDを複数選択できる列です。
右隣のボタンから編集できます。
IDと値のペアリスト 他のデータベースのIDと、関連値を複数選択できる列です。
右隣のボタンから編集できます。
ピクチャー 画像を選択できる列です。アイコンを選択する場合はシステムDBで指定されたアイコンファイルが使われます。
プログラム側では、選択された画像ファイルのパスが文字列値として保存されます。
アイコンを選択した場合は、アイコングラフィック内の位置情報が文字列値として保存されます。
曲線データ 必要な情報（基底値・係数・指数）を編集して曲線を生成します。
右隣のボタンから編集できます。
アドオン アドオン情報を編集します。
右隣のボタンから編集できます。
SQボタン 付随するスクリプトを開いて編集します。
このボタンを押すことで、選択した項目のFixedIDに名前のコメントを添えたテキストデータがクリップボードにセットされます。

DB: システム

ゲームプログラム全般の設定です。ただし、一部はエディターでしか使われない設定も含んでいます。
このデータベースだけは内部的な設定の名前が公開されていて、特殊な編集方法になっています。
設定値が不正であるときはエラーメッセージが表示されます。

ファイルを指定する設定項目について

ファイルを指定するダイアログでは自由にファイルを選択できるようになっていますが、実際にはプロジェクトフォルダー内のファイルのみ有効となりますのでご注意下さい。

ユニット指標の計算式について

いわゆる「戦闘力」を表す値の計算式を定義できます。
スクリプトのように計算式をそのまま記述できます。
式中には優先順位を高める括弧 ( ) や 基本的な演算子 + - * / が使えます。
割り算については、整数で割ってしまうと正しく計算されないことがあるため、小数点以下の数を付けるようにして下さい。

所持金 単一値 初期状態の所持金です。

DB: ユニット

ユニットとは、複数のエネミーを一つのグループとしてまとめた、実際に戦闘として遭遇する単位のことを指します。
このデータベースは表形式ではなくグラフィカルに編集します。

指標について

システムDBで設定した計算式に基づいて各エネミーの指標（いわゆる「戦闘力」に相当）を計算し、それらの和をユニットの指標とします。
この「指標」は基本的には「強さ」を示すことを想定しており、ゲームバランスの調整に役立てる目的で設けています。

登録リスト

ユニットとして登録されているエネミーのリストです。

デザイナー

バトル画面上で表示される各エネミーの位置をドラッグ＆ドロップで編集します。

背景変更

デザイナー上に表示されるサンプル用の背景画像を設定します。
データとしては保存されませんので、実際の戦闘画面には反映されませんのでご注意下さい。

バトルテスト

登録したユニットとの戦闘テストを行います。
パーティメンバー編成とそれぞれのメンバーのレベルを設定して開始します。
パーティメンバーの装備品は常に初期装備となります。

ユニットスクリプトを編集

ユニットのスクリプトを開いて編集します。
戦闘システムによって用途や役割は大きく変わりますが、プリセットのスクリプトではイベントのように戦闘の各種フェーズの都度呼び出される仕組みになっています。

列情報

内部的には他のデータベースと同様にTSV形式で保存されますので、ここではその内部的な列情報を示します。

配置情報 文字列値 ｛エネミーID;表示位置X座標;表示位置Y座標|...｝の形式で示される配置情報です。

DB: エフェクト

スキルやアイテムを使用したときに表示される演出効果です。
このデータベースは表形式ではなくグラフィカルに編集します。
ここで編集したエフェクト固有の命令情報は、エディターによって付随スクリプトとして読み書きされます。
このスクリプトを実行することでエフェクトが再生できる仕組みになっています。

フレームリスト

画面左部の # から始まる番号でリストされた部分です。選択しているフレームが編集対象となります。

フレーム情報パネル

現在のフレームの描画情報に関わる設定を編集するパネルです。

  • 区画座標 … レイヤーパレット上のセクション単位の位置 (X, Y) です。
  • 透明度 … 0-255 の間で指定します。
  • 拡大率 … パーセント値で指定します。等倍は 100 となります。
  • 回転角度 … 度数で指定します。

レイヤーパレット

現在編集対象となっているレイヤーのグラフィックが表示されており、区画座標を選択します。

（現在の）レイヤー

編集対象とするレイヤーを選択します。先頭に付加される番号はレイヤーのインデックスを表します。

命令リスト

指定したフレームのタイミングで実行される各種命令です。

  • SEを再生する … 汎用素材データベースのサウンドサブDBに登録されている効果音を再生します。
  • 画面の減衰振動 … 指定した強さで画面全体を振動させます。
  • 画面のフラッシュ … 指定した色、強さ、回数で画面全体にフラッシュ（点滅させる）効果を施します。
  • 同期再生の待機を解除 … そのフレームに達した時点で再生待ちをしている呼び出し元のスクリプトに処理を返します。
  • 独自コマンドを呼び出す … 任意のスクリプトコードを実行します。

デザイナー

表示位置をドラッグ＆ドロップで編集します。編集対象のレイヤー以外は操作できません。

ツールバー：描画基準

エフェクトが複数同時に表示されるとき、どのように表示されるかを示します。

対象個別 それぞれのアクターに対して一つずつ表示します。
対象一括 対象のサイド（パーティ・ユニット）に対して一つだけ表示します。
画面中心 画面中心に一つだけ表示します。

ツールバー：フレーム数

エフェクトの再生時間（フレーム数）を変更します。

ツールバー：レイヤー設定

エフェクトのグラフィック（レイヤー）を変更します。
複数のグラフィックを使うことができます。
各レイヤーには区画サイズ（1コマあたりのサイズ）を設定します。

ツールバー：ターゲット

デザイナー上に表示させるサンプルプレビュー用のエネミーグラフィックを変更します。

ツールバー：再生

デザイナー上でエフェクトをプレビューします。プレビュー時は市松模様とグリッド線が非表示になります。

ツールバー：レイヤーシフト

編集対象のレイヤーの全フレームに対して、指定ピクセル分だけ表示座標を移動させます。

ツールバー：直前フレーム情報を引き継ぐ

編集対象のフレームの一つ前のフレーム情報を取り込みます。

ツールバー：フレームをコピー

現在のフレーム情報を、指定したフレーム範囲に対してコピーします。

ツールバー：フレームをクリア

現在のフレーム情報を初期化します。

ツールバー：フレーム一括

指定したフレーム範囲に対して、始点フレームの情報を一括で適用します。
コピーと異なるのは適用する項目を選択できるという点です。

ツールバー：フレーム補間

指定したフレーム範囲に対して、始点フレームから終点フレームにかけて線形的に補間します。
一連のコマが並べられたレイヤーグラフィックを扱う場合は、始点と終点に区画座標を設定するだけで簡単にアニメーション化することができます。

独自コマンドの編集

エフェクトの命令で任意のスクリプトコードを呼び出す際、外部のスクリプトを取り込んで使う場合に編集します。
再生の都度コンパイルが行われるため、項目を増やせば増やすほどその分処理負荷が上がります。

列情報

内部的には他のデータベースと同様にTSV形式で保存されますので、ここではその内部的な列情報を示します。

フレーム数 単一値 このエフェクトのフレーム数です。
描画対象 単一値 ツールバー「描画対象」で設定した項目のインデックスです。
レイヤー情報 文字列値 ｛レイヤーID:レイヤーのグラフィックファイル名:セクションサイズ|...｝の形式で示されるレイヤー情報です。

DB: タイルセット

マップで使われるタイルパレットとして使われるものです。
このデータベースは表形式ではなくグラフィカルに編集します。
ここで編集したタイルセット個別の情報は、エディターによって別途タイル情報定義ファイルとして読み書きされます。

オートタイルリスト

オートタイルとして認識させるグラフィックを設定します。最大で127個まで登録できます。
タイルごとに各種設定（当たり判定等）を施すので、並び替えると各種設定も一緒に移動します。

オブジェクトタイルリスト

オブジェクトタイルとして認識させるグラフィックを設定します。個数に上限はありません。
タイルごとに各種設定（当たり判定等）を施すので、並び替えると各種設定も一緒に移動します。

デザイナー：当たり判定

そのタイルの上を歩けるかどうかを設定します。
当たり判定は上層から順に判定され、空白タイルもしくは「下位レイヤーに依存」であるときは次のレイヤーを判定します。
すべてのレイヤーが「下位レイヤーに依存」するときは、通行できるものとします。

デザイナー：出入設定

そのタイルの上に入るとき、指定した方向に対してのみの出入りを許可します。
タイルの角をクリックするとまとめて2つの方向の切り替えが行えます。
タイルの中心をクリックすると、全方向の切り替えが行えます。
出入設定は上層から順に判定され、空白タイルもしくは「下位レイヤーに依存」であるときは次のレイヤーを判定します。
また、オートタイルに限り移動元と移動先が同じタイルであるときは出入設定に関わらず移動できるものとします。

デザイナー：描画優先度

タイルが描画される順序を設定します。以下のような設定があります。

  • 通常描画
  • そのタイルに重なったキャラを隠す（キャラを裏に回す）
  • 重なったキャラの下半身を半透明表示
  • 最前面表示

デザイナー：グループ番号

タイルに対して任意の番号を振ることができます。
ここで振った番号の用途はスクリプトに依存します。プリセットのスクリプトでは、グループ番号: 99番 が梯子に相当します。
左クリックで１つ加算、右クリックで１つ減算、中央クリックで直接番号を入力できます。

内部的には他のデータベースと同様にTSV形式で保存されますので、ここではその内部的な列情報を示します。

メモ 文字列値 プリセットのスクリプトでは使われていませんので、自由なメモとしてお使い下さい。
オートタイルリスト 文字列値 ｛ファイル名|...｝の形式で示されるオートタイルのファイルリストです。

DB: 汎用素材

ゲームプログラムを起動したとき、タイトル画面に入る前に読み込まれる素材です。
頻繁に使われる素材は直前のロード時間を短縮するためにゲーム開始時にロードしておくことをお勧めします。
ただし、これらの項目が多ければ多いほど、起動に時間がかかるようになります。
ここで読み込む素材の用途はすべてスクリプトに依存します。

フェイスグラフィック

メッセージ表示に伴って表示させるキャラクターの画像です。

グラフィック ピクチャー 読み込む画像です。
反対側 論理値 プリセットのスクリプトでは通常、左側にフェイスを表示します。
これをキャラクターの向きの都合で反対側に表示させたい場合に有効にします。

汎用グラフィック

一般的な描画に使える画像です。

グラフィック ピクチャー 読み込む画像です。

汎用サウンド

主に効果音として使用するサウンドです。
BGMやBGSはサイズが大きく、読み込みに時間がかかりメモリ消費を増やすことになるのでなるべく控えて下さい。
どうしても登録が必要な場合は、何度も頻繁に使用する共通的なものに限定することをお勧めします。

ファイル名 文字列値 読み込むサウンドです。

汎用フォント

フォント正式名称 文字列値 フォントの名前です。
無効なフォント名を指定した場合は既定の「ＭＳ ゴシック」（ＭＳは全角、スペースは半角）となります。
フォントファイル名 文字列値 外部のフォントを使う場合は、ここにファイル名を指定します。
大きさ 単一値 フォントのドット数です。
太さ 単一値 フォントの太さです。既定値は６です。
行間 単一値 改行したときに追加するピクセル数です。０のときフォントサイズ分に相当します。
文字間 単一値 文字と文字の間に追加するピクセル数です。

DB: 基本要素

パラメーター

パーティキャラやエネミーに対して設定されるパラメーターです。

属性

アクターに対して紐付く属性です。

スキル種別

スキルに対して設定される種別です。

アイテム大種別

アイテムに対して設定される種別です。たとえば「消耗品」や「武器」等がこちらに相当します。

装備品 論理値 この大種別が設定されたアイテムが装備品であるかどうかを示します。

アイテム小種別

アイテム大種別をさらに細分化する種別です。たとえば、武器に対する「剣」や「斧」等に相当します。
ただし、このデータベースでアイテム大種別と関連付けることはありません。

DB: ユーザー定義

組み込みのデータベースとは一線を画し、列名や列の型も含めてすべて自由に編集できるデータベースです。
作成した個々のデータベースは、別途TSVファイルとして読み書きされます。
また、データベースの選択を切り替えた時点で即時保存される特殊な仕組みになっています。

FixedIDについて

ユーザーデータベース内では、各行に対してFixedIDが割り当てられません。
しかし、左端の列の型を「単一値」とすることでスクリプトからID（その列の値）を使った操作ができるようになります。
このとき、もし値が重複していた場合は先に見つかった方（インデックスが若い方）の項目が操作対象となります。

列の追加と編集

列種別で示した型に対応した列を自由に設定できます。
また、値を格納しないボタン列も編集することができます。
ボタン列については、特定のセルに挿入するための補助を行うものになります。
その代入先を指定するときは、左端を 0 として数えたインデックスを指定します。
